Extract Hive Query Statement from a Hive Script

Basic Question

It's a real problem to solve. In one of the projects in our team, we need to use HiveParser to parse some hive scripts. But before that, we need to extract statements from a hive query script.

That's what you need to do: follow the syntax rules below to extract statements from a script.

Problem Definition

What is a statement?

Hive script is a plain text file which contains one or multiple statements. Each statement is terminated by ;.

There are many types of statement:

1) Set statement, the statement following the pattern SET <parameter_name>=<parameter_value>;

For example,

SET mapreduce.job.running.map.limit=1;

is a set statement

2) Query statement, all remaining statement can be defined as a query statement

Your program will not be tested against an invalid Hive script, so please do not spend your precious time trying to validate the statements.

Syntax rule

Comment

A comment can appear anywhere in a statement. You can include a comment in a statement in two ways:

1) Begin the comment with a slash and an asterisk /*. Proceed with the text of the comment. This text can span multiple lines. End the comment with an asterisk and a slash */. The opening and terminating characters need not be separated from the text by a space or a line break.

For example,

/* select id
  from table
  where column = "a" */

is a comment.

2) Begin the comment with two hyphens --. Proceed with the text of the comment. This text cannot extend to a new line. End the comment with a line break.

For example,

-- select id  from table where column = "a"; \n

is a comment.

String

Everything in "" or '' should be regarded as a string, which means even though a comment is inside the quotes, it will be still regarded as a string.

For example,

'-- select id  from table where column = "a" \n'

is not a comment but valid string.

Another example,

select id  from table where string_column = '-- select id  from table where column = "a" \n';

is a valid select statement.

What your program does:

  1. Removes all the set statements and comments from the script.
  2. Prints the rest of the statements to the console.
  3. Again, it's beyond the scope of this test to validate the statement printed out.

What is a hint?

You can use comments in a SQL statement to pass hints, to help Hive query optimizer. The optimizer uses these hints as suggestions for choosing an execution plan for the statement.

A statement block can have only one comment containing hints, and that comment must follow the SELECT, UPDATE, INSERT, or DELETE keyword. The following syntax shows hints contained in both styles of comments that Hive supports within a statement block.

{DELETE|INSERT|SELECT|UPDATE} /*+ hint [text] [hint[text]]... */

or

{DELETE|INSERT|SELECT|UPDATE} --+ hint [text] [hint[text]]...

For example,

select /*+ MAPJOIN(time_dim) PREPAREDATA(mobile.store_sales) */ count(*) from
store_sales join time_dim on (ss_sold_time_sk = t_time_sk)

Hint is MAPJOIN(time_dim) and PREPAREDATA(mobile.store_sales)

Please note:

  • DELETE, INSERT, SELECT, or UPDATE are the keywords that begins a statement block. Comments containing hints can appear only after these keywords.
  • + is a plus sign that causes Hive to interpret the comment as hint. The plus sign must follow immediately after the comment delimiter (no space is permitted).
  • The space between the plus sign and the hint is optional.
  • If the comment contains multiple hints, then separate the hints by at least one space.
